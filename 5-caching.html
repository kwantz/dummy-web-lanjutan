5. Caching

Kita menggunakan webpack untuk memaketkan file-file ke dalam folder ./dist. Saat semua file yang ada didalam ./dist berjalan pada server, dari sisi client akan mengambil file-file js dan css yang ada dibutuhkan. Karena memakan waktu untuk dapat membuka semua file tersebut, browser menggunakan teknik caching supaya sebuah website berjalan dengan cepat karena kurangnya trafik network yang tidak dibutuhkan. Tetapi bagi para developer, ini merupakan sebuah masalah yang memunculkan sakit kepala (Iya, saya juga sakit kepala karena saat ingin mencoba semua code diatas, saya harus men-clear semua cache yang ada di browser setiap terjadi pergantian code.) Panduan ini berfokus pada konfigurasi yang diperlukan untuk memastikan file yang dihasilkan oleh webpack dapat tetap di-cache kecuali isinya telah berubah.
