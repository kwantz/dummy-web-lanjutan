1. Konsep pada Webpack

Webpack adalah pembundel modul statis untuk aplikasi JavaScript modern. Ketika webpack memproses aplikasi, secara internal dibuat grafik ketergantungan yang memetakan setiap modul yang dibutuhkan proyek tersebut dan menghasilkan satu atau lebih bundel. Sejak versi 4, webpack tidak memerlukan file konfigurasi untuk membundel proyek, namun dapat dikonfigurasi untuk disesuaikan dengan kebutuhan kita. Ada beberapa konsep inti dari webpack yaitu :

Entry

Entry point menunjukkan modul webpack mana yang harus digunakan untuk mulai membangun grafik ketergantungan internal, webpack akan mencari tahu modul dan library lain yang mana titik masuk bergantung (secara langsung maupun tidak langsung). Secara default, nilainya adalah ./src/index.js, tapi kita dapat menentukan yang lain (atau banyak entry point) dengan mengkonfigurasi properti entry di konfigurasi webpack seperti :

Output

      
Properti output memberitahu webpack di mana untuk meletakkan bundel yang dibuatnya dan bagaimana menamai file-file tersebut, yang secara default ./dist/bundle.js adalah file output utama dan folder ./dist sebagai tempat file lain dihasilkan. Loader

      
Berbeda dengan yang lain, webpack hanya memahami file JavaScript. Loader memungkinkan webpack untuk memproses jenis file lain dan mengubahnya menjadi modul valid yang dapat dikonsumsi oleh aplikasi kita dan ditambahkan ke grafik dependensi. Sebagai catatan, kemampuan untuk mengimpor semua jenis modul misalnya file .css adalah fitur khusus dari webpack dan mungkin tidak dimiliki oleh bundler lainnya. Pada penggunaan lebih lanjut, Loader memiliki dua fungsi dalam konfigurasi webpack:

Plugin

      
Sementara loader digunakan untuk mengubah jenis modul tertentu, plugin dapat dimanfaatkan untuk melakukan berbagai tugas yang lebih luas, mulai dari optimisasi bundel, manajemen aset ataupun menyuntikkan variabel lingkungan. Untuk menggunakan plugin, harus di require() dan tambahkan ke array plugin. Kebanyakan plugin dapat disesuaikan melalui opsi. Karena sebuah plugin dapat digunakan beberapa kali dalam sebuah konfigurasi untuk tujuan yang berbeda, kita perlu membuat turunannya dengan menggunakan operator new.

      

      
Dari contoh di atas, plugin akan menghasilkan sebuah file html dari aplikasi yang secara otomatis menyuntikkan semua bundel yang dihasilkan.


      
Mode

      
Dengan mengatur parameter mode baik untuk development, production atau none, kita dapat mengaktifkan pengoptimalan bawaan webpack yang sesuai dengan setiap lingkungan. Nilai default adalah production.
