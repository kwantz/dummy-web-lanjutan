Loader

Loader adalah transformasi yang diaplikasikan pada source code dari sebuah modul. Loader memungkinkan kita untuk memproses file sebelum dimpor atau dimuat. Dengan demikian, loader menyediakan cara untuk menangani langkah-langkah membangun front-end. Loader dapat mengubah file dari bahasa yang berbeda (seperti TypeScript) ke dalam JavaScript, atau gambar inline sebagai URL data. Loader bahkan memungkinkan kita untuk mengimpor file CSS langsung dari modul JavaScript.

Ada 3 cara untuk menggunakan loader di aplikasi kita :

1. Konfigurasi (direkomendasikan) : Tuliskan pada file webpack.config.js. module.rules memungkinkan kita untuk menentukan beberapa loader dalam konfigurasi webpack. Ini adalah cara singkat untuk menampilkan loader, dan membantu menjaga kerapian kode. Kita juga mendapat gambaran lengkap dari masing-masing loader. 



      

        2. Inline : Tuliskan secara jelas di setiap import.
        Adalah mungkin untuk menuliskan loader di sebuah pernyataan impor, atau metode pengimpor yang setara.
        Pisahkan loader dari sumber dengan ! (tanda seru). Setiap bagian relatif terhadap direktori saat ini.
        Options dapat disampaikan dengan parameter query atau dengan sebuah objek JSON.
      

      

      

        Contoh di atas ialah penulisan ulang kembali contoh pada bagian Konfigurasi dimana options modules dituliskan dalam bentuk query.
        Untuk options yang dituliskan dalam bentuk objek JSON dapat dilihat pada contoh berikut ini.
      

      


      

        3. CLI : Tuliskan dalam bentuk perintah shell.
        Kita juga dapat menggunakan loader dengan CLI.
        Untuk contoh pada bagian konfigurasi,
        berikut ialah cara penulisannya dalam CLI.
      

      

      


      
Fitur-fitur yang tersedia pada loader:

      
    
        
  • 
          Loader dapat dirantaikan. Mereka diaplikasikan dalam bentuk saluran ke sumber.
          Serantai loader dijalankan dengan urutan terbalik. Loader pertama dalam rantai loader mengembalikan nilai ke nilai
          berikutnya. Pada loader terakhir, webpack mengharapkan JavaScript yang dikembalikan.
        
    • 
        
  • Loader dapat berupa sync atau async.
    • 
        
  • Loader berjalan di Node.js dan dapat melakukan semua hal yang mungkin di sana.
    • 
        
  • Loader menerima parameter query. Ini dapat digunakan untuk mengirim konfigurasi ke loader.
    • 
        
  • Loader juga dapat dikonfigurasi dengan sebuah objek options.
    • 
        
  • Modul normal dapat mengekspor loader selain dari main secara normal melalui package.json dengan field loader.
    • 
        
  • Plugin dapat memberikan lebih banyak fitur kepada loader.
    • 
        
  • Loader dapat menghasilkan file tambahan yang tidak diperlukan.
